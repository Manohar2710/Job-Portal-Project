Step 1 — Set Up a Postman Environment

Create an environment named job-portal-local with these variables. Postman scripts will auto-populate the token fields after each request.

2a

URL

Scripts Tests tab (auto-saves tokens)
// Paste this in the Tests tab of the Register request
const res = pm.response.json();
pm.environment.set("accessToken",  res.accessToken);
pm.environment.set("refreshToken", res.refreshToken);

Both tokens are returned and saved to the environment automatically.

Step 3 — Call a Protected Endpoint

3
Use the access token on any protected API
GET
URL (example)
{{baseUrl}}/api/jobs/1
Authorization tab
FieldValue
TypeBearer Token
Token{{accessToken}}
Postman will inject Authorization: Bearer <accessToken> into the request header automatically.
200 OK
Token is valid — you get the job data.
401
Token is expired — proceed to Step 4 to refresh it.

Step 4 — Refresh the Access Token (the key step)

4
Exchange refresh token for new token pair
POST
URL
{{baseUrl}}/api/auth/refresh
No Authorization header needed. The refresh token travels in the request body, not as a Bearer token. Leave the Authorization tab set to "No Auth".
Body · raw · JSON
{
  "refreshToken": "{{refreshToken}}"
}
Scripts Tests tab — IMPORTANT: replaces both tokens
// The old refreshToken is now dead. Save the new pair immediately.
const res = pm.response.json();
pm.environment.set("accessToken",  res.accessToken);
pm.environment.set("refreshToken", res.refreshToken);
200 OK
New access token (24 h) + new refresh token (7 d) returned. Old refresh token is deleted from DB.
401
Refresh token not found or expired — user must log in again.
400
refreshToken field is blank or missing.

Step 5 — Logout

5
Invalidate the refresh token server-side
POST
URL
{{baseUrl}}/api/auth/logout
Body · raw · JSON
{
  "refreshToken": "{{refreshToken}}"
}
Scripts Tests tab — clear saved tokens
pm.environment.unset("accessToken");
pm.environment.unset("refreshToken");
204 No Content
Refresh token deleted from DB. No response body.

Negative Tests — Verify Security Behaviour

Run these after Step 4 to confirm the rotation and expiry logic is working correctly.

!
Replay the OLD refresh token after rotation
POST

After a successful /refresh, copy the old refresh token string from Postman's console history, then send it again to /api/auth/refresh.

401 Expected
Token was deleted during rotation — cannot be replayed. ✓
!
Use refresh token after logout
POST

Call /api/auth/logout, then immediately call /api/auth/refresh with the same token.

401 Expected
Token was deleted on logout — server rejects it. ✓
!
Send a random garbage token
POST
{
  "refreshToken": "this-is-not-a-real-token"
}
401 Expected
Not found in DB — rejected. ✓

Tips

ScenarioWhat to do
Want to test an expired access token without waiting 24 h Temporarily change security.jwt.expiration in application.yaml to 60000 (1 minute), restart the app, login, wait 1 min, then call a protected endpoint.
Want to test an expired refresh token Change security.jwt.refesh-expiration to 60000 (1 minute), restart, login, wait 1 min, then call /refresh. Expect 401.
See the token payload Paste the accessToken JWT value into jwt.io to inspect sub, roles, iat, exp claims. The refresh token is opaque — nothing to decode.
Automate the full flow Create a Postman Collection and chain the requests using the Tests scripts above. Use "Run Collection" to execute all steps in order.
